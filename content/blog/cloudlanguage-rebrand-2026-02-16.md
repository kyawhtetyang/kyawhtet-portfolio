CloudLanguage - Burmese-First Learning MVP
Project Diary

This started from a real family need. My mom wanted to learn English so she could teach her 4-year-old grandchild, William. But most language apps assume English as the default and there was no Burmese-first path to learn English. That pushed me to build CloudLanguage.

I rebranded the early Duolingo-inspired prototype into CloudLanguage and shipped a mobile-first MVP in about 1.5 hours using Codex. I handled frontend, backend, and VPS deployment in that single sprint. It was the fastest full-stack build I have ever done.

The app is different from typical language apps. It feels like listening to music: you hear audio while the “lyrics” (translation and pronunciation cues) update in real time. That concept pushed me to design a backend data structure that aligns tightly with the frontend, reducing storage complexity and making the UI flow clean.

This project also made me realize a big wave of software disruption is coming, and speed + clarity of system design will matter even more.

The current focus is mobile-first web. Next steps include a native mobile app, recommendation systems like Spotify or YouTube, and training a voice model from our own recordings using a supervised pipeline (likely on Colab).

Skills gained: rapid MVP delivery, mobile-first product design, backend data modeling aligned to UI, and VPS deployment under tight time constraints.
